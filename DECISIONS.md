| 2026-04-19 | AGE loader uses direct INSERT into `ag_catalog."{graph}"."{label}"` tables instead of `cypher() UNWIND` + `%s` JSON parameter | Keep `cypher() UNWIND` and fix psycopg2 parameter binding; use Apache AGE Python driver (registers agtype adapter); use PREPARE/EXECUTE to force server-side Param nodes | psycopg2's `execute(query, params)` uses libpq's simple query protocol, which substitutes `%s` parameters as SQL string literals client-side. AGE's `cypher()` function enforces at parse time that its third argument is a PostgreSQL `Param` node (server-side bind parameter). String literals fail this check with `InvalidParameterValue: third argument of cypher function must be a parameter`. Direct INSERT into AGE's internal vertex and edge tables is a standard bulk-load technique that bypasses `cypher()` entirely; regular SQL `%s::agtype` casts work without the Param-node constraint. Additional AGE-specific discoveries: (1) table reference must be `"{graph}"."{label}"` not `ag_catalog."{graph}"."{label}"` even with `ag_catalog` on search_path; (2) agtype property access uses `agtype_to_text(properties -> '"id"')` not `->>` operator; (3) graphid values must be passed as `%s::agtype::graphid`, not plain Python ints. |
| 2026-04-19 | Edge loader resolves CURIE endpoints to AGE graphids via an in-memory `curie_to_id` dict built by querying all vertex label tables after node load | Look up graphid per-edge via individual SELECT (slow for 693M edges); use a temporary join table in Postgres; skip endpoint resolution and rely on AGE MATCH for edge inserts | After `load_nodes()` completes, all vertex tables are populated. A single pass over all 10 vertex label tables reads `(id, agtype_to_text(properties -> '"id"'))` and builds a dict in memory. At Phase 4 scale this holds ~115M CURIE->graphid pairs (~8-10 GB RAM, same budget as merger.py's dedup set). The in-memory lookup costs O(1) per edge during the 693M-edge insert pass vs O(log N) or O(N) for a per-edge SELECT. The dict is built in `pipeline.py` between node and edge load steps and passed to `load_edges()` to avoid rebuilding it for every batch. |
