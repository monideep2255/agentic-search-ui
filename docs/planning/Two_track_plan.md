# Agentic search: two-track plan

Two parallel tracks. One feeds the other. Neither depends on the other.

---

## Track 1: personal build (portfolio project)

Goal: get reps extracting data, building pipelines, understanding what works and what doesn't. This becomes a portfolio piece.

What it is: open source agentic search across NCBI databases, built on personal time and dime.

Key docs (in Data/):
- Agentic_search_architecture_QA.md: architecture blueprint (read first)
- Personal_build_plan.md: 2-month execution plan (read second)

Reference material (in Data/reference/):
- NCBI_databases_and_APIs_reference.md: raw API data for all 39 databases
- KG_prototype_feedback.md: SME feedback defining query patterns and quality bar
- Decision_log_April_2_2026.md: brainstorming directions from April 2 session
- Vision_of_success.md: personal north star for evaluating the project

Status: architecture brainstormed (April 2), not yet validated or committed to. Next step is reviewing the architecture QA and agreeing on what to actually build.

---

## Track 2: innovation proposal (NCBI submission)

Goal: get NCBI funding for an official agentic search project.

What it is: formal innovation proposal submitted through NCBI's process. Separate scope, separate timeline, separate audience.

Key docs (in Proposals/):
- Innovation_proposal_2026.md: the proposal itself
- Agentic_search_vision_goals_canvas.md: Confluence canvas version
- Feedback_tracker.md: all feedback sources and their status

Status: v2 edits complete. Remaining: deep read, convert to .docx, update Confluence. Deadline is Monday April 6 (submit to Anne).

---

## How they relate

Track 1 learnings inform Track 2, but Track 2 does not wait for Track 1.

- If the innovation proposal gets accepted: Track 1 gives you a head start on understanding what works, what breaks, and how to revise the architecture for production.
- If the innovation proposal does not get accepted: Track 1 is your portfolio project regardless. Open source, publishable, career asset.

The architecture decisions in Track 1 (Data folder) are brainstorming, not commitments. The innovation proposal (Proposals folder) has its own scope and framing.
