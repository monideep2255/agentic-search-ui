## Decisions made

1. System 3 vision confirmed: user asks a question, agent queries across three data layers through hard guardrails, returns cited answer. Over time, user interactions become competency questions that feed back into the orchestrator.

2. Competency question routing: start with few-shot examples in the planner prompt. Do not build a classification layer or fine-tuned model until we have enough real query data to justify it.

3. Planning before building: create a PRD and technical specification before writing any System 3 code. These become the shared source of truth for both human and agent.

4. Team model: no human team coming. Build an agent army instead. Map every product/engineering role to an agent workflow. Monideep orchestrates.

5. Hard stop rule: no PRD/tech spec updates mid-build. New information from KGC, Nodes AI, or other sources gets parked in the reference folder and evaluated for v2.

6. New information flow: collect to reference folder, index in Background_requirements.md, evaluate post-v1.

---

## Key discussion points

Readiness assessment: all infrastructure is in place (docs, dependencies, config, knowledge graph live, git clean). No blockers. The gap is implementation code, which is intentional since we plan before we build.

Competency questions: Monideep raised that calling all three layers on every query creates unacceptable latency. Competency questions solve this by teaching the orchestrator which layers to hit for which query shapes. Three mechanisms discussed (few-shot, classification layer, fine-tuned model). Agreed to start simple (few-shot) and let data guide the upgrade path.

Why plan first: Monideep's approach is to discuss everything, make decisions, then execute cleanly. Agent context resets between conversations, so the PRD and tech spec also serve as shared memory between sessions.

Agent army: not getting a human team, so building agent workflows for every role (researcher, developer, tester, reviewer). Bossman-mode skill is the execution framework. Update it as we learn what works.
